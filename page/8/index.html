建站记录

这次的建站还是比较成功的，页面我也很喜欢。

首先说一下这次冲动的原因。其实之前一直就想要有个博客的，因为这一阵子做了很多东西本以为学了很多，没想到再次遇到同样的问题的时候还是得回去看原来的代码，所以体会到了整理总结并且记录下来的重要性。而且你看那些网站一个个的跟我要个人网站地址，但是我又没有，VPS是有的，域名也是有的，干脆一不做二不休建一个博客吧。

其实在准备建站之前也去看了看CSDN的博客，还在上面写了篇文章，但是每次都要审核好麻烦的，而且CSDN有些界面还是那么的丑，然后去SegmentFault上面也看了看那个专栏，并且把之前在CSDN上写的那篇文章复制过去了，我还好心的在后面写了一句“复制自我刚写的CSDN博客”，然后审核就不通过了，原因是“文章质量不高”，我本来以为是我写的太零碎了，没想到还真是因为我的最后一句注释，因为我把它删掉之后再次提交审核竟然通过了，由此我就很不开心。。。

所以我昨天晚上开始建站，因为喜欢上了MarkDown，所以在找能用md的博客，不经意间发现了GitBlog，发现这个很适合小巧的VPS，所以我就决定尝试一下。

一开始都很顺利，安装nginx，安装php的一些组件，之后配置nginx，然后把GitBlog放到nginx配置的网站根目录下，结果发现主页什么都不显示，然后我就加了GitBlog的交流群，群主很热心的为我解答一些问题，然后还是不行，我就把我的vps帐号给了群主，他帮我检查（那时候已经凌晨一点了），最后检查出来是因为缺少了mbstring扩展库，他帮我安装好了，真的是很感谢群主。

到此为止我的博客算是建立成功了，之后就是最为重要的记录了。

现在是研零的暑假，马上就要开学成为真正的研究生了，我要养成一个记录已经学到的知识的好习惯。与GitBlog共勉。

GitNote:PATH问题和安装路径问题

前几天为了免于每次登陆VPS都要输密码做了一些尝试，其中尝试过使用”_netrc”文件的方法，这个方法要新建一个HOME环境变量，值为”%USERPROFILE%”，刚新建好之后还没什么问题，重启了之后再次使用就有问题了。

如果我从开始菜单里面打开git bash，那么没有问题，如果我在项目的文件夹上面右键然后选择git bash的话，就会出一系列的问题，比如用VI编辑完文件（或者只是查看文件）要退出的时候，第一次输入完退出命令之后，bash会提示无法往viminfo写入,具体错误信息如下：

$ $PATH
bash:/d/MyProject/ModBus/%USERPROFILE%/bin:/mingw64/bin:/usr/local/bin:/usr/bin:/bin:/mingw64/bin:/usr/bin:%USERPROFILE%/bin:/c/WINDOWS/system32:/c/WINDOWS:/c/WINDOWS/System32/Wbem:/c/WINDOWS/System32/WindowsPowerShell/v1.0:/c/Program: No such file or directory
```

我删除HOME环境变量后卸载git，然后重装，然后卸载，然后安装低版本，然后循环了三遍还是不行。关机重启之后再重新安装git就好了，所以我想Windows的环境变量重启之后才会有实质变化？

然后还有一个问题就是如果把git按照默认路径安装到C:/Program Files/Git之后如果使用git config --system指令的话可能会提示Permission denied，需要把Git安装目录的权限改一下才行，要给当前用户写入权限。

当然如果一开始就安装在了非系统盘的话就没有这个问题。
