用VS组建一个Qt多工程的解决方案来进行软件开发

一直在各种场合看到真正的Visual Studio软件工程项目，无一不是多工程协同工作的，或者说多模块合作编程，不同的模块分为不同的子工程生成DLL，这样更新子工程的时候只需要更新DLL即可，无需重新交付用户新的版本，这样听起来就很有吸引力，这才是真正的软件工程师该干的事情，我终于得到机会亲自尝试一把了，这让我在软件工程师的道路上走了一大步。

当我把架构设计初稿用Atash UML完成之后，就准备着手实际新建一个这样的VS工程了，这期间遇到了不少问题，有一些也是很有意义的问题，是启发思维的问题，于是记录一下。那么首先说一下重构的软件架构，典型的MVC式的结构。下面就是简单的结构图。

这个系统结构简图忽略了项目特有的一些特征。可以看到主要有三层，底层是Database,SystemIO,SystemTCP三个，其中SystemTCP是SystemIO的一部分，再往上是Controller，在上面就是负责显示的子工程SystemView，除了SystemView是生成可执行文件以外，其他子工程都是生成DLL，他们按照箭头逆方向依赖，SystemView依赖于Controller，也就是说说SystemView要调用Controller的DLL，其余部分相类似。

那么我们就可以到Visual Studio里面新建工程了，新建五个工程（当然要先新建一个工程，然后再工程的解决方案中添加新的工程），然后VS也提供了文件夹功能，你可以使用解决方案中的文件夹让整个解决方案看起来符合逻辑结构。

新建完工程之后就需要让VS知道他们之间的关系，首先是项目依赖关系，也就是编译顺序，这个是在解决方案的属性中设置的（右键解决方案，选择属性，然后在通用属性中找到第二个项目依赖，按照实际的依赖关系选择即可，比如Controller依赖Database和SystemIO，SystemIO依赖SystemTCP这样）。这样设置完之后可以编译下试试，可以发现编译顺序就是按照设定的顺序来的。

当然，这还没完，这样只是设置了编译顺序，实际上各个项目之间半毛钱关系都没有，需要在每个工程里面添加它依赖的工程的引用目录、库目录、附加依赖项，这个可以参照我之前的那篇博文《如何调试一个链接库工程工程》。但是那篇博客里面只是说了要添加这些， 但是怎么添加路径这个事情其实是有大学问的。

假设你把绝对路径作为参数添加进去之后，实际使用起来没有任何问题，但是问题就是，你从Debug换到Release的时候，就会发现库目录这个东西不对了，因为之前添加的是调试DLL的目录，所以肯定就不行了。这种问题其实其他程序员前辈们早就考虑好了，要知道程序员可能是世界上最懒的工种了，解决办法就是路径Macro，当你添加的时候你会在界面上发现有一个Macro按钮，点开你就会发现所有的路径宏。如下图：

另外附上几个常用的路径宏：

class $PROJECTNAME_EXPORT ClassName
{
public:
ClassName();
~ClassName();
};
```

否则会出现连接错误。

如果不加这个可以把动态链接库改成静态链接库也可以，但是加上这个之后就不能改静态链接库了。

还有非常重要的一点就是父工程需要包括所有的子工程的引用目录和Qt Project Settings，比如一个子工程用到了SQL，那么所有的父工程都要包含SQL，否则可能出现打不开头文件的连接错误。

Oracle数据库&远程连接&ODBC驱动

因为高级数据库课程需要，开始学习Oracle数据库，学习Oracle首先要安装，我尝试在linux上安装，结果失败了，然后把笔记本重新安装成Windows之后在上面安装上了Oracle。（在Windows上面安装Oracle也会提示很多错误，全部忽略就行了）

Oracle安装完成之后，可以通过SQLplus进行连接，但是sqlplus不是图形化界面不利于保存SQL语句，还是想要通过图形界面访问，一开始使用PLSQL，但是一直连接不上，后来直接用Oracle提供的sqldeveloper进行连接。

本地连接没什么特殊的，很容易就连接成功了，让我头疼的是如何远程连接，我按照网络上的教程，使用Net Managment配置本地服务和监听器，但是在远端使用Sqldeveloper就是通过不了测试，后来发现网上有教程说我缺少Instant Client（即时客户端），下载之后配置好了发现也不行。

我突然想起了之前sqlserver远程连接的时候需要重启服务，于是我去重启了一遍Oracle的TNSListener服务，果真，重启之后配置重新载入就好了。

Oracle小记：

  1. 每个Oracle用户一个表方案，访问不同用户的表需要在表名字前面加用户名。
  2. Oracle新建数据库是新建一个服务。
  3. create user *name* identified by *passwd*创建用户
  4. grant connect,dba,resource to *name*分配权限

在Windows上安装Oracle驱动

为了做分布式实验，需要在一个没有安装Oracle数据库的Windows上面用ODBC来访问oracle，百度了很多，那些回答、博客都跟傻逼似的，一点用也没有，唯一有用的是这个，这篇博客说的是对的。

所以我们需要下载两个东西一个是base包(instantclient-basic)，另一个是ODBC包(instantclient-odbc)，下载下来之后是两个压缩包，解压然后合并（其实都把它们解压到一个位置即可，因为里面的文件夹名字都是一样的，会自动合并），然后把这个文件夹放置到随便一个你喜欢的地方，然后进去用管理员权限运行”odbc_install.exe”就行了，这时候ODBC管理程序里面就会出现Oracle in instantclient了。

要连接服务器还必须得配置TNS，在instantclient根目录下面新建一个目录叫做”NETWORK”，进去再新建一个目录叫做”ADMIN”，在ADMIN里面新建一个文件叫做”tnsnames.ora”（当然你也可以从Oracle服务器里面直接复制出来），这时候还需要配置两个环境变量，分别是ORACLE_HOME和NLS_LANG。第一个变量值就是instantclient目录（比如D:\instantclient_11_2），第二个变量值等于SIMPLIFIED CHINESE_CHINA.ZHS16GBK，这个变量是解决sql通过odbc查询中文乱码问题的。

这时候你去ODBC管理程序里面就可以配置Oracle服务器了。TNS Service Name里面可以下拉出你在tnsnames.ora配置的内容（如果下拉不到东西说明你ORACLE_HOME变量没有配置）。

后来我发现……想要在SqlServer Management Studio里面连接Oracle数据库最简单的方法还是在电脑上安一个Oracle客户端……

最后还是没能逃过安装Oracle……

void MyTCP::SendRequest(qint8 slaveaddress, qint8 functioncode, qint16 startaddress, qint16 datanumber)
{
QByteArray line;
QDataStream out(&line, QIODevice::WriteOnly);
out.setVersion(QDataStream::Qt_DefaultCompiledVersion);
out << qint8(00) << qint8(00) << qint8(00) << qint8(00);//protocal identifier 4byte
qint16 messagelen = sizeof(slaveaddress) + sizeof(functioncode) + sizeof(startaddress) + sizeof(datanumber);
out << messagelen;//High 8-bit ModBus & Low 8-bit message length
out << slaveaddress;//slave address
out << functioncode;//function code
out << startaddress;//High 8-bit & Low 8-bit of start address
out << datanumber;//High 8-bit & Low 8-bit of data number
tcpSocket.write(line);

}

void MyTCP::ReadData()
{
QDataStream in(&tcpSocket);
in.setVersion(QDataStream::Qt_DefaultCompiledVersion);
qint64 availabelnum = tcpSocket.bytesAvailable();
QByteArray res = tcpSocket.readAll();
}
```

##QDate：
QDate是Qt提供的日期控件可以获取当前日期，如果是QDateTime的话可以精确到秒。获取了日期之后可以设置日期显示格式。由toString(QString format)函数实现设置格式。

这个toString函数类似printf函数，都是把format参数里面的特定字符替换为时间，比如y代表year，M代表month，由于日期时间格式里面有两个“m”，所以大写M表示month，小写m表示minute，d代表day，h代表hour，s代表second,z代表毫秒。举个例子：

那么返回就是2015-08-22-16-33；
2015.10.07补充（摘自互联网）：
这些是可能用到的日期表达式：

d - 没有前置0的数字的天（1－31）
dd - 前置0的数字的天（01－31）
ddd - 缩写的日名称（Mon－Sun）。使用QDate::shortDayName()。
dddd - 长的日名称（Monday－Sunday）。使用QDate::longDayName()。
M - 没有前置0的数字的月（1－12）
MM - 前置0的数字的月（01－12）
MMM - 缩写的月名称（Jan－Dec）。使用QDate::shortMonthName()。
MMMM - 长的月名称（January－December）。使用QDate::longMonthName()。
yy - 两位数字的年（00－99）
yyyy - 四位数字的年（0000-9999）

这些是可能用到的时间表达式：

h - 没有前置0的数字的小时（0－23或者如果显示AM/PM时，1－12）
hh - 前置0的数字的小时（00－23或者如果显示AM/PM时，01－12）
m - 没有前置0的数字的分钟（0－59）
mm - 前置0的数字的分钟（00－59）
s - 没有前置0的数字的秒（0－59）
ss - 前置0的数字的秒（00－59）
z - 没有前置0的数字的毫秒（0－999）
zzz - 前置0的数字的毫秒（000－999）
AP - 切换为AM/PM显示。AP将被“AM”或“PM”替换。
ap - 切换为am/pm显示。ap将被“am”或“pm”替换。
所有其他输入字符都将被忽略。

QFileDialog是一个可以获取文件路径的类，使用它会自动弹出一个文件选择对话框。这个类同时也可以获取文件夹路径。

Gitblog&CuteMarkED快速添加博文前缀

自从搭建起Gitblog之后就一直在找寻更加便捷的写博文和发博文的方法，昨天实现了自动发布博文之后，今天也实现了一种更加便捷的添加博文前缀的方法。

感谢github中的CuteMarkED工程，我在这个工程的基础之上添加了一个往文件中添加gitblog前缀的功能，以下是使用帮助：

CuteMarkEd的其他用法请参照CuteMarkEd的Github主页。

  • 在菜单栏的最后面有一个gitblog菜单，点击Profix，将会弹出如图界面

  • 在界面中可以填写作者、标题、标签、总结等，可以选择状态和已有文集，时间默认为当前日期，如果希望调整可以自行调整。

  • 如果想要添加文集的话可以点击添加文集按钮，会出现如图的对话框：

    这个时候只需要输入想要的文集名称然后点击PushButton即可（竟然忘了改按钮名字！），然后文集列表中就会出现你刚刚添加的文集名称。如果想要删除文集的话，可以到本软件的安装目录或者解压目录下找到categories.txt打开并删除想要删除的文集即可。

  • 填写完内容之后点击OK按钮，博文前缀就会添加到当前编辑框的最前面。

  • 另外，如果直接双击打开.markdown或者.md文件的话，在打开Profix对话框时程序会检查md文件目录下有无categories.txt文件，如果没有就会显示文件无法打开的错误。可以手动添加一遍或者把程序根目录下的文件拷贝过去均可。
    建议把所有的md文件都集中在同一个文件夹下。如果是通过快捷方式或者直接打开CuteMark程序的话就没有这个问题。

我打包好了这个软件的安装包和解压包，有兴趣的朋友可以从这里下载。
